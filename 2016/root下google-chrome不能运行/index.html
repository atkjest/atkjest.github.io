root下google-chrome不能运行

发表于2016-03-02
默认 小 中 大

google-chrome为了安全考虑，拒绝在root用户下执行，在终端运行google-chrome会发现什么也不会输出，谷歌到的好多答案都是gedit /usr/bin/google-chrome，然后在文件的末尾添加--user-data-dir，即exec -a "$0" "$HERE/chrome" "$@" --user-data-dir，添加之后还是不能启动。

最后搜索到把谷歌的沙箱sandbox关了就可以了，以google-chrome %U --no-sandbox启动就可以了，最后修改启动文件gedit /usr/bin/google-chrome
如下：

if [[ -n "$CHROME_USER_DATA_DIR" ]]; then
  # Note: exec -a below is a bashism.
  exec -a "$0" "$HERE/chrome"  \
    --user-data-dir="$CHROME_USER_DATA_DIR" "$@"
else
  exec -a "$0" "$HERE/chrome"  --user-data-dir="/root/.config/google-chrome/"  \
    "$@" --no-sandbox
fi
